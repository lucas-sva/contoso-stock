# Contributing Guidelines

Thank you for considering contributing to this project!  
These guidelines will help you understand how to collaborate effectively.

## How to Contribute

1. **Fork the repository**  
   Create your own copy of the project.

2. **Create a branch**  
   Use a descriptive name for your branch:  
   `git checkout -b feature/add-new-endpoint`

3. **Make your changes**  
   - Follow the coding style used in the project.  
   - Add or update tests when relevant.  
   - Update documentation if needed.

4. **Commit your changes**  
   Write clear and concise commit messages:  
   `git commit -m "Add endpoint for order creation"`

5. **Push your branch**  
   `git push origin feature/add-new-endpoint`

6. **Open a Pull Request (PR)**  
   - Describe the changes you made.  
   - Reference related issues if applicable.  
   - Be open to feedback and discussion.

## Code Style

- Use meaningful variable and function names.  
- Keep functions small and focused.  
- Write tests for new features and bug fixes.  

## Reporting Issues

- Use the **Issues** tab to report bugs or suggest features.  
- Provide as much detail as possible (steps to reproduce, screenshots, logs).  
- Be respectful and constructive.

## Community Standards

This project follows the [Code of Conduct](CODE_OF_CONDUCT.md).  
Please make sure your contributions align with these values.
